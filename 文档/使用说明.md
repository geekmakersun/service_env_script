## 快速开始

### 环境部署流程

按照以下顺序执行脚本，完成 LNMP 环境部署：

```bash
cd /root/服务脚本库/环境脚本

# 1. 安装 Nginx + ModSecurity
chmod +x 1. Nginx+ModSecurity安装脚本.sh
./1. Nginx+ModSecurity安装脚本.sh

# 2. 安装 MariaDB
chmod +x 2. MariaDB安装脚本.sh
./2. MariaDB安装脚本.sh

# 3. 安装 PHP 8.4 FPM
chmod +x 3. PHP84-FPM安装脚本.sh
./3. PHP84-FPM安装脚本.sh

# 4. 使用站点管理工具创建网站
./4. Nginx站点管理工具.sh
```

## 脚本说明

### 1. Nginx+ModSecurity安装脚本.sh

**功能**: 从源码编译安装 Nginx 1.25.4 + ModSecurity 3.0.12

**特点**:
- 自动下载并编译 ModSecurity v3
- 集成 Nginx-ModSecurity 连接器
- 配置自定义安全规则
- 支持自定义错误页面
- 自动创建目录结构和权限

**配置变量**:
```bash
NGINX_VERSION="1.25.4"              # Nginx 版本
MODSECURITY_VERSION="3.0.12"        # ModSecurity 版本
MODSECURITY_NGINX_VERSION="1.0.3"   # 连接器版本
INSTALL_PREFIX="/etc/nginx"         # 安装路径
```

**查看详细文档**: [Nginx-ModSecurity-部署指南.md](./Nginx-ModSecurity-部署指南.md)

### 2. MariaDB安装脚本.sh

**功能**: APT 方式安装 MariaDB 10.11+

**特点**:
- 彻底清理之前的安装残留
- 交互式安全配置

**执行流程**:

## 服务管理命令

### Nginx

```bash
# 查看状态
systemctl status nginx

# 启动/停止/重启
systemctl start nginx
systemctl stop nginx
systemctl restart nginx

# 平滑重载配置
systemctl reload nginx
# 或
nginx -s reload

# 测试配置
nginx -t

# 查看错误日志
tail -f /var/log/nginx/error.log
```

### MariaDB

```bash
# 查看状态
systemctl status mariadb

# 启动/停止/重启
systemctl start mariadb
systemctl stop mariadb
systemctl restart mariadb

# 登录数据库
mysql -u root -p

# 查看版本
mysql -V
```

### PHP-FPM

```bash
# 查看状态
systemctl status php84-fpm

# 启动/停止/重启
systemctl start php84-fpm
systemctl stop php84-fpm
systemctl restart php84-fpm

# 平滑重载
systemctl reload php84-fpm

# 查看 PHP 版本
php -v

# 查看已加载的扩展
php -m
```

## 目录说明

### Nginx 目录

| 路径 | 说明 |
|------|------|
| `/etc/nginx/` | Nginx 主配置目录 |
| `/etc/nginx/enabled/` | 启用的站点配置 |
| `/etc/nginx/disabled/` | 禁用的站点配置 |
| `/etc/nginx/conf.d/` | 附加配置文件 |
| `/etc/nginx/snippets/` | 共用配置片段 |
| `/etc/nginx/modsecurity/` | ModSecurity 配置和规则 |
| `/etc/nginx/ssl/` | SSL 证书目录 |
| `/var/log/nginx/` | Nginx 日志目录 |
| `/var/www/` | 网站根目录 |

### MariaDB 目录

| 路径 | 说明 |
|------|------|
| `/etc/mysql/` | 配置文件目录 |
| `/var/lib/mysql/` | 数据文件目录 |
| `/var/log/mysql/` | 日志目录 |
| `/var/run/mysqld/` | Socket 和 PID 文件 |

### PHP 目录

| 路径 | 说明 |
|------|------|
| `/usr/local/php84/` | PHP 安装目录 |
| `/usr/local/php84/etc/` | 配置文件目录 |
| `/usr/local/php84/etc/php-fpm.d/` | FPM 池配置 |
| `/var/log/php-fpm/` | PHP-FPM 日志 |
| `/var/run/php-fpm/` | Socket 文件目录 |

## 常用配置

### Nginx 默认配置

主配置文件: `/etc/nginx/nginx.conf`

```nginx
user www-data;
worker_processes auto;
error_log /var/log/nginx/error.log warn;
pid /var/run/nginx.pid;

events {
    worker_connections 1024;
    use epoll;
    multi_accept on;
}

http {
    include /etc/nginx/mime.types;
    default_type application/octet-stream;
    
    log_format main '$remote_addr - $remote_user [$time_local] "$request" '
                    '$status $body_bytes_sent "$http_referer" '
                    '"$http_user_agent" "$http_x_forwarded_for"';
    
    access_log /var/log/nginx/access.log main;
    
    sendfile on;
    tcp_nopush on;
    tcp_nodelay on;
    keepalive_timeout 65;
    
    gzip on;
    gzip_vary on;
    gzip_proxied any;
    gzip_comp_level 6;
    gzip_types text/plain text/css text/xml application/json application/javascript;
    
    # 包含站点配置
    include /etc/nginx/enabled/*;
}
```

### PHP-FPM 默认配置

主配置文件: `/usr/local/php84/etc/php-fpm.conf`

池配置: `/usr/local/php84/etc/php-fpm.d/www.conf`

```ini
[www]
user = www-data
group = www-data
listen = /var/run/php-fpm/php84-fpm.sock
listen.owner = www-data
listen.group = www-data
pm = dynamic
pm.max_children = 50
pm.start_servers = 5
pm.min_spare_servers = 5
pm.max_spare_servers = 35
```

## 故障排查

### 查看服务状态

```bash
# 查看所有服务状态
systemctl status nginx mariadb php84-fpm

# 查看失败的服务
systemctl list-units --type=service --state=failed

# 查看系统日志
journalctl -xe
```

### 常见问题

**问题1: 502 Bad Gateway**

原因: PHP-FPM 未运行或 socket 配置错误

解决:
```bash
# 检查 PHP-FPM 状态
systemctl status php84-fpm

# 检查 socket 文件
ls -la /var/run/php-fpm/

# 重启 PHP-FPM
systemctl restart php84-fpm
```

**问题2: 无法连接数据库**

原因: MariaDB 未运行或认证问题

解决:
```bash
# 检查 MariaDB 状态
systemctl status mariadb

# 本地免密登录
mysql -u root

# 检查用户权限
SELECT User, Host FROM mysql.user;
```

**问题3: 权限错误 (403)**

原因: 网站目录权限不正确

解决:
```bash
# 修复权限
chown -R www-data:www-data /var/www/example.com
chmod -R 755 /var/www/example.com
```

## 安全建议

### 1. 防火墙配置

```bash
# 安装 UFW
apt install ufw

# 默认拒绝所有入站
ufw default deny incoming

# 允许 SSH
ufw allow 22/tcp

# 允许 HTTP/HTTPS
ufw allow 80/tcp
ufw allow 443/tcp

# 启用防火墙
ufw enable
```

### 2. 定期更新

```bash
# 更新系统
apt update && apt upgrade -y

# 更新 Nginx（源码安装需手动更新）
# 参考 Nginx-ModSecurity-部署指南.md

# 更新 MariaDB
apt update mariadb-server mariadb-client

# 更新 PHP（源码安装需手动更新）
# 参考 PHP-8.4-FPM-编译安装指南.md
```

### 3. 备份策略

```bash
# 备份 Nginx 配置
tar -czvf nginx-backup-$(date +%Y%m%d).tar.gz /etc/nginx/

# 备份网站文件
rsync -avz /var/www/ /backup/www/

# 备份数据库
mysqldump -u root -p --all-databases > all-databases-$(date +%Y%m%d).sql
```
