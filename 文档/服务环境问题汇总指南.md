## 重启后 502 Bad Gateway 错误

### 问题描述
服务器重启后，网站无法访问，出现 502 Bad Gateway 错误。

### 问题原因
重启后以下服务无法正常启动：

1. **PHP 8.4-FPM** - 缺少 `/var/run/php-fpm` 临时目录
2. **MariaDB** - 无法创建 `/var/run/mariadb` 临时目录
3. **服务冲突** - 同时存在 `php-fpm.service` 和 `php84-fpm.service` 两个服务

### 解决方案

#### 1. 创建临时目录
```bash
mkdir -p /var/run/php-fpm /var/run/mariadb
chown -R www-data:www-data /var/run/php-fpm
chown -R mysql:mysql /var/run/mariadb
```

#### 2. 停止冲突服务
```bash
systemctl stop php-fpm.service
systemctl disable php-fpm.service
```

#### 3. 重启核心服务
```bash
systemctl restart mariadb.service
systemctl restart php84-fpm.service
```

#### 4. 长期解决方案
已创建 `create-temp-dirs.service` 系统服务，确保每次重启时自动创建临时目录。

### 验证服务状态
```bash
# 检查所有关键服务状态
systemctl status mariadb.service php84-fpm.service nginx.service

# 检查失败的服务
systemctl list-units --type=service --state=failed
```

### 临时目录位置
- PHP-FPM: `/var/run/php-fpm/`
- MariaDB: `/var/run/mariadb/`

## MariaDB 访问被拒绝错误

### 问题描述
```
(conn=4) Access denied for user 'root'@'localhost' (using password: YES)
```

### 问题原因
- `root@localhost` 原本使用 `mysql_native_password` 认证且无密码
- 某个应用尝试用密码连接 localhost，导致认证失败

### 解决方案
已配置为使用 `unix_socket` 认证，本地通过 socket 连接无需密码。

## MariaDB 安全配置

### 当前安全配置（推荐）
已配置为最高安全性：
- **仅保留 `root@localhost`** - 使用 unix_socket 认证（本地免密码）
- **删除了 `root@127.0.0.1` 和 `root@::1`** - 禁止通过 TCP/IP 连接
- **MariaDB 仅监听 127.0.0.1** - 不暴露在公网

### 本地免密码连接
```bash
# 本地连接（无需密码，通过 Unix socket）
/usr/local/mariadb10.11/bin/mysql -u root

# 或使用 mysql 命令
mysql -u root
```

### 查看认证配置
```bash
/usr/local/mariadb10.11/bin/mysql -u root -e "
SELECT Host, User, JSON_EXTRACT(Priv, '$.plugin') as plugin 
FROM mysql.global_priv 
WHERE User='root';
"
```

### 查看监听地址
```bash
# 确认 MariaDB 只监听本地
ss -tlnp | grep 3306
```

### 重新配置安全认证
如果需要重新配置安全认证，执行以下 SQL：

```sql
-- 删除不必要的 root 用户
DROP USER IF EXISTS 'root'@'127.0.0.1';
DROP USER IF EXISTS 'root'@'::1';

-- 确保 root@localhost 使用 unix_socket
UPDATE mysql.global_priv 
SET Priv = JSON_SET(Priv, '$.plugin', 'unix_socket')
WHERE User='root' AND Host='localhost';

FLUSH PRIVILEGES;
```

## MariaDB 数据库初始化失败

### 问题描述
在编译安装 MariaDB 10.11.16 后，运行初始化脚本时出现以下错误：

```
FATAL ERROR: Could not find /usr/local/mariadb10.11//usr/local/mariadb10.11/share/fill_help_tables.sql
```

或

```
FATAL ERROR: Could not find .//usr/local/mariadb10.11/bin/my_print_defaults
```

或

```
ERROR 1146 (42S02) at line 1: Table 'mysql.db' doesn't exist
```

### 问题原因

1. **路径重复问题**：`mariadb-install-db` 脚本在检测 `basedir` 时，如果指定了 `--basedir` 参数，会错误地拼接路径，导致路径重复（如 `/usr/local/mariadb10.11//usr/local/mariadb10.11/`）

2. **缺少必要工具**：`my_print_defaults` 工具位于编译构建目录的 `extra` 子目录中，脚本无法找到该工具

3. **数据库未正确初始化**：系统表不存在，导致后续操作失败

### 解决方案

#### 1. 停止所有 mysqld 进程

```bash
sudo pkill -9 mysqld
sleep 2
```

#### 2. 清空数据目录

```bash
sudo rm -rf /data/mysql/*
```

#### 3. 使用正确的参数初始化数据库

```bash
sudo /usr/local/mariadb10.11/bin/mariadb-install-db \
    --no-defaults \
    --user=mysql \
    --srcdir=/root/mariadb/src/mariadb-10.11.16 \
    --builddir=/root/mariadb/src/mariadb-10.11.16/build \
    --datadir=/data/mysql
```

**重要参数说明**：
- `--no-defaults`: 不读取任何配置文件
- `--user=mysql`: 以 mysql 用户身份运行
- `--srcdir`: 指定 MariaDB 源码目录
- `--builddir`: 指定编译构建目录（包含 my_print_defaults 等工具）
- `--datadir`: 指定数据目录

**注意事项**：
1. **不要使用 `--basedir` 参数**：会导致路径重复错误
2. **必须同时指定 `--srcdir` 和 `--builddir`**：脚本需要从这两个目录找到必要的文件
3. `--builddir` 必须指向编译构建目录（`/root/mariadb/src/mariadb-10.11.16/build`）

#### 4. 设置数据目录权限

```bash
sudo chown -R mysql:mysql /data/mysql
```

#### 5. 启动服务

```bash
sudo systemctl start mariadb
```

#### 6. 验证安装

```bash
# 检查服务状态
sudo systemctl status mariadb

# 测试连接（root 用户无密码）
mysql -u root -e "SELECT VERSION(); SHOW DATABASES;"
```

### 常见错误及解决方法

#### 错误 1: FATAL ERROR: Could not find fill_help_tables.sql

**错误信息**：
```
FATAL ERROR: Could not find /usr/local/mariadb10.11//usr/local/mariadb10.11/share/fill_help_tables.sql
```

**原因**：使用了 `--basedir` 参数，导致路径重复

**解决**：不要使用 `--basedir` 参数，改用 `--srcdir` 和 `--builddir`

#### 错误 2: FATAL ERROR: Could not find my_print_defaults

**错误信息**：
```
FATAL ERROR: Could not find .//usr/local/mariadb10.11/bin/my_print_defaults
```

**原因**：脚本无法找到 `my_print_defaults` 工具

**解决**：使用 `--builddir` 参数指定编译构建目录

#### 错误 3: Can't lock aria control file

**错误信息**：
```
[ERROR] mariadbd: Can't lock aria control file '/data/mysql/aria_log_control' for exclusive use, error: 11
```

**原因**：数据目录被另一个 mysqld 进程锁定

**解决**：停止所有 mysqld 进程后重新初始化

#### 错误 4: ERROR 1146 (42S02): Table 'mysql.db' doesn't exist

**错误信息**：
```
ERROR 1146 (42S02) at line 1: Table 'mysql.db' doesn't exist
```

**原因**：数据库未正确初始化，系统表不存在

**解决**：清空数据目录后使用正确的参数重新初始化

### 脚本修复

已修复 `MariaDB安装脚本.sh` 中的初始化部分，将原来的 `mysql_install_db` 替换为 `mariadb-install-db`，并使用正确的参数：

```bash
# 使用 mariadb-install-db 初始化数据库
log_info "使用 mariadb-install-db 初始化系统表..."
${INSTALL_PREFIX}/bin/mariadb-install-db \
    --no-defaults \
    --user=mysql \
    --srcdir=/root/mariadb/src/mariadb-${MARIADB_VERSION} \
    --builddir=/root/mariadb/src/mariadb-${MARIADB_VERSION}/build \
    --datadir=${DATA_DIR} \
    2>&1 | tee -a "$LOG_FILE"

chown -R mysql:mysql ${DATA_DIR}

log_success "数据库初始化完成"
```
