# 单一网址SSL证书申请脚本设计文档

## 核心原则

> **最小化修改原则**：脚本只添加、修改必要的内容，尽可能不乱改配置。

| 操作 | 说明 |
|------|------|
| ✅ 允许 | 申请证书、存储证书、替换证书路径 |
| ✅ 允许 | 检查并修复密钥配置文件权限 |
| ❌ 禁止 | 修改监听端口、server_name、root等配置 |
| ❌ 禁止 | 添加或删除其他Nginx指令 |
| ❌ 禁止 | 修改HTTP/2、HTTP/3、QUIC等协议配置 |

---

## 一、需求背景

为Nginx站点配置SSL证书时，需要一个简洁的工具来处理单一网址的证书申请场景。

### 目标场景

Nginx配置文件中 `server_name` 包含类似以下内容：
```nginx
server_name sub.example.com;
```

需要实现：
- 直接为 `sub.example.com` 这个子域名申请证书
- 不涉及主域名 `example.com`
- 证书目录名称与网址一致

---

## 二、功能需求

### 2.1 核心功能

| 功能 | 说明 |
|------|------|
| 配置文件识别 | 根据传入的配置文件路径，自动读取 server_name |
| 域名单一申请 | 只为 server_name 中的第一个域名申请证书 |
| 证书存储 | 证书目录名与申请的域名完全一致 |
| 自动配置 | 自动更新Nginx配置文件中的SSL路径 |
| 测试申请 | 正式申请前自动执行测试申请，防止浪费申请次数 |

### 2.2 输入参数

```bash
./3.单域名证书申请.sh <配置文件路径> [--dns验证方式]
```

**参数说明：**

| 参数 | 必填 | 说明 |
|------|------|------|
| 配置文件路径 | 是 | Nginx站点配置文件的完整路径 |
| --aliyun-dns | 否 | 使用阿里云DNS验证（推荐，无需修改配置） |
| --cloudflare-dns | 否 | 使用Cloudflare DNS验证（推荐，无需修改配置） |

**说明：** 脚本会自动在正式申请前执行测试申请（不计入速率限制），无需手动指定测试参数。

**验证方式对比：**

| 验证方式 | 是否需要停止Nginx | 是否修改配置 | 适用场景 |
|----------|-------------------|--------------|----------|
| HTTP验证（默认） | 是 | 否 | 无DNS API密钥时使用 |
| 阿里云DNS验证 | 否 | 否 | 域名托管在阿里云 |
| Cloudflare DNS验证 | 否 | 否 | 域名托管在Cloudflare |

## 三、配置文件示例

### 3.1 输入配置文件

```nginx
server {
    listen 80;
    server_name api.example.com;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl;
    server_name api.example.com;

    root /var/www/api.example.com/public;
    index index.php index.html;

    ssl_certificate /etc/nginx/ssl_certs/old.example.com/fullchain.pem;
    ssl_certificate_key /etc/nginx/ssl_certs/old.example.com/privkey.pem;
}
```

### 3.2 执行命令

```bash
./3.单域名证书申请.sh /etc/nginx/enabled/api.example.com.conf
```

### 3.3 输出配置文件

```nginx
server {
    listen 80;
    server_name api.example.com;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl;
    server_name api.example.com;

    root /var/www/api.example.com/public;
    index index.php index.html;

    ssl_certificate /etc/nginx/ssl_certs/api.example.com/fullchain.pem;
    ssl_certificate_key /etc/nginx/ssl_certs/api.example.com/privkey.pem;
}
```

**说明**: 仅替换了 `ssl_certificate` 和 `ssl_certificate_key` 路径，其他配置保持不变。

### 3.4 证书存储结构

```
/etc/nginx/ssl_certs/
└── api.example.com/
    ├── 20240101_120000/
    │   ├── fullchain.pem
    │   ├── privkey.pem
    │   ├── cert.pem
    │   └── chain.pem
    ├── latest -> 20240101_120000/
    ├── fullchain.pem -> latest/fullchain.pem
    └── privkey.pem -> latest/privkey.pem
```

## 四、错误处理

### 4.1 配置文件不存在

```bash
错误: 配置文件 /etc/nginx/enabled/xxx.conf 不存在
请检查文件路径是否正确
```

### 4.2 无法提取域名

```bash
错误: 无法从配置文件中提取 server_name
请确保配置文件包含有效的 server_name 指令
```

### 4.3 证书申请失败

```bash
错误: 证书申请失败
可能原因:
1. 域名DNS未正确解析到本服务器
2. 防火墙阻止了80/443端口
3. Let's Encrypt速率限制
4. DNS验证密钥无效

建议:
- 检查域名解析: dig api.example.com
- 检查端口开放: netstat -tlnp | grep -E '80|443'
- 查看详细日志: /var/log/letsencrypt/letsencrypt.log
```

### 4.4 速率限制触发

```bash
警告: 检测到可能触发Let's Encrypt速率限制
当前域名: api.example.com
近期申请次数: 4/5 次
距离重置时间: 3天12小时

建议:
1. 脚本会自动执行测试申请（不计入限制）
2. 等待速率限制重置后再申请
3. 使用已存在的有效证书

是否继续申请? [y/N]
```

### 4.5 证书已存在且有效

```bash
信息: 证书已存在且有效
域名: api.example.com
证书路径: /etc/nginx/ssl_certs/api.example.com/
有效期至: 2024-04-01 12:00:00
剩余天数: 45天

跳过证书申请，直接配置Nginx...
```

## 五、依赖文件

| 文件 | 用途 |
|------|------|
| `密钥配置/aliyunak.conf` | 阿里云DNS验证密钥 |
| `密钥配置/cloudflare.conf` | Cloudflare DNS验证密钥 |
| `./1.Nginx重启.sh` | Nginx重启脚本 |

### 5.1 密钥配置文件处理

脚本启动时会自动检查密钥配置目录和文件：

```
密钥配置/
├── aliyunak.conf          # 阿里云DNS验证密钥（权限: 600）
└── cloudflare.conf        # Cloudflare DNS验证密钥（权限: 600）
```

**处理逻辑：**

1. **目录检查**
   - 检查 `密钥配置/` 目录是否存在
   - 不存在则自动创建

2. **文件检查**
   - 检查密钥配置文件是否存在
   - 存在则验证权限是否为 `600`
   - 权限不正确则自动修复

3. **权限要求**
   - 密钥文件包含敏感信息，必须设置为 `600`（仅所有者可读写）
   - 目录权限设置为 `700`（仅所有者可访问）

**示例输出：**

```bash
检查密钥配置目录...
目录已存在: 密钥配置/
检查阿里云密钥配置...
文件已存在: 密钥配置/aliyunak.conf
权限检查: 当前 644, 需要 600
修复权限: chmod 600 密钥配置/aliyunak.conf
权限已修复
```

## 六、速率限制防护机制

### 6.1 Let's Encrypt 限制规则

| 限制类型 | 限制值 | 重置周期 |
|----------|--------|----------|
| 每个域名证书申请次数 | 5次/周 | 7天 |
| 每个账户注册域名数 | 300个/周 | 7天 |
| 验证失败次数 | 5次/小时 | 1小时 |
| 重复证书申请 | 5次/周 | 7天 |

### 6.2 防护策略

1. **证书存在性检查**
   - 检查 `/etc/nginx/ssl_certs/域名/` 目录
   - 检查 `/etc/letsencrypt/live/域名/` 目录

2. **证书有效性检查**
   - 解析证书过期时间
   - 计算剩余有效期
   - 有效期 > 30天时跳过申请

3. **测试申请机制**
   - 正式申请前自动执行测试申请
   - 使用 Let's Encrypt 测试环境（--test-cert）
   - 不计入正式申请次数限制
   - 测试成功后自动清理测试证书
   - 测试失败则终止，不进行正式申请

4. **申请记录追踪**
   - 记录每次申请时间到 `密钥配置/.申请记录`
   - 格式: `域名|申请时间|申请结果`
   - 用于估算速率限制状态

### 6.3 申请记录文件格式

```
# 文件: 密钥配置/.申请记录
# 格式: 域名|时间戳|结果|证书路径
api.example.com|1704067200|success|/etc/nginx/ssl_certs/api.example.com/
test.example.com|1704153600|failed|-
api.example.com|1704240000|success|/etc/nginx/ssl_certs/api.example.com/
```
